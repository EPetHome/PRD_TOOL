# 三分钟上手指南

> 版本:V1.0
> 受众:第一次接触这套范式的同事 / PM

---

## 这套范式是什么、为什么存在

一个 AI 产品从"想法"到"可交付"需要三类产出:**原型**(给工程师看的可点击页面)、**文档**(给所有人类看的产品说明)、**AI 构建包**(给 AI 工具看的结构化指令)。这三类产出物面向三类不同受众,**必须物理分开**,才能各自做到位 —— 这就是本套件的底层设计。

本套件提供 6 份文件,PM 复制到自己项目里,**一天内**能独立产出三模块合规交付物,并额外生成一份综合评审页。

---

## 三模块对照表

| 模块 | 受众 | 产出形态 | 用套件里哪份文件 |
|---|---|---|---|
| **模块 1 · 原型** | 算法 / 前端 / 后端工程师 | 可双击打开的单文件 HTML 原型 | 《模块 1 · 原型脚手架》 |
| **模块 2 · 文档** | 所有人类(产品 / 运营 / 法务 / 客户方等) | 叙事性 Markdown PRD | 《模块 2 · 文档模板》 |
| **模块 3 · AI 构建包** | AI 工具(Cursor / Claude Code / v0 / 通用大模型) | 结构化 Markdown | 《模块 3 · 产出物规范》+《模块 3 · 转换 Prompt》 |
| **综合评审页** | 评审会 / 汇报 / 跨团队对齐 | 原型 + PRD 合并 HTML | 《模块 1+2 · 综合评审页脚手架》 |

## PM 工作流(6 步 · 1 天内完成)

### 第 1 步 · 想清楚产品是什么(0.5-1 小时)

开始填写之前,自己先回答:

- 产品给谁用?解决什么问题?
- 核心功能 3-5 个是哪些?
- 商业形态是什么(MVP / 内测 / 规模化 / 成熟期)?

**想不清楚 → 停下,找利益相关方对齐,不要硬填。**

### 第 2 步 · 填写模块 2 文档(2-4 小时)

打开《模块 2 · 文档模板.md》,复制到自己项目目录下,重命名为 `[[ 产品名 ]]-模块 2-文档.md`,按模板填空,填完**删掉所有 `[[ ]]` 占位符**。其中第 8 章"下游转译输入"必须填,它会直接供模块 1 原型和模块 3 AI 构建包抽取。

填写结束后,**按模板附录 A 自查清单逐项勾选**。未过清单不得提交评审。

### 第 3 步 · 模块 2 人工评审(0.5-1 小时)

让**至少一位非技术同事**(运营 / 法务 / 其他 PM)通读,确认"能完整复述产品做什么"。评审通过后,模块 2 就是交付状态。

**评审不通过 → 回第 2 步修改,不要强推。**

### 第 4 步 · 生成模块 3(0.5 小时)

打开《模块 3 · 转换 Prompt.md》,复制"Prompt 正文"整段到任一 AI 工具(Claude / ChatGPT / Cursor / Claude Code 等),**在 Prompt 后紧接着粘贴评审通过的模块 2 文档全文**,发送。

AI 产出 AI 构建包。先按《模块 3 · 转换 Prompt.md》里的"核对清单"核 5 分钟,再用同文件里的"可执行性验证 Prompt"检查它是否能直接交给 Cursor / Claude Code / v0 开工。两项都通过后,保存为 `[[ 产品名 ]]-AI 构建包.md`。

**核对不通过 → 优先回模块 2 第 8.5 补齐技术选型,或按核对清单提示改模块 2 重跑,不要强交付。**

### 第 5 步 · 生成模块 1 原型(0.5 小时)

打开《模块 1 · 原型脚手架.md》,复制其中"Prompt 骨架"整段到任一 AI 工具,**在 Prompt 后紧接着粘贴评审通过的模块 2 文档全文**,发送。

模块 1 会同时按**AI 产品经理视角**和**专业交互设计师视角**规划原型:既保证核心功能 / AI 决策 / 异常态完整,也保证移动端信息层级、主操作和四态反馈清楚。验收时重点看右侧手机原型是否能**自己点击走通主流程**,左侧控制台只应切换主要演示变量,不能替代真实页面跳转。

AI 产出 HTML 代码块。把代码块另存为 `[[ 产品名 ]]-原型.html`,**双击用浏览器打开**,按脚手架里的"核对清单"核 5 分钟,核对通过就是交付状态。

**核对不通过 → 优先回模块 2 第 8.1-8.4 补齐业务主变量 / 输入项 / 输出结果 / 状态语义,再重跑 Prompt。**

### 第 6 步 · 生成综合评审页(0.5 小时)

打开《模块 1+2 · 综合评审页脚手架.md》,复制其中"Prompt 骨架"整段到任一 AI 工具,然后依次粘贴:

1. 评审通过的模块 2 文档全文
2. 已验收通过的模块 1 原型 HTML 全文

AI 产出综合评审页 HTML。把代码块另存为 `[[ 产品名 ]]-综合评审页.html`,双击打开。它应该是三栏:左侧切换台,中间可交互手机原型,右侧可阅读 PRD 正文和目录。

**核对不通过 → 优先检查是否误把"切换台 + 手机"一起塞到左侧、手机原型是否在中栏首屏独立出现、右侧 PRD 是否只是 Markdown 原文、TOC 是否可跳转。**

## 什么情况下该停下找人

- **产品定位没想清楚**(第 1 步卡住) → 找产品负责人或业务方对齐
- **模块 2 评审不通过**(第 3 步卡住) → 回第 2 步改,别忽略反馈
- **AI 产出模块 3 一直有"待产品方补充"项**(第 4 步卡住) → 模块 2 第 8 章或第 5 章有缺口,回第 2 步补齐
- **模块 1 原型 AI 怎么都做不对**(第 5 步卡住) → 模块 2 的核心流程 / 第 8 章转译输入没讲清楚,回第 2 步补齐
- **综合评审页中栏手机原型空白、手机被塞到左栏下方,或右侧 PRD 不可读**(第 6 步卡住) → 重新用《模块 1+2 · 综合评审页脚手架》生成,不要手拼
- **法规不确定**(模块 2 第 7 章卡住) → 找法务,不要自己拍脑袋
