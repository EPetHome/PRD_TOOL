# 模块 1+2 · 综合评审页脚手架

> 版本:V1.1
> 受众:PM(需要把原型和人读 PRD 放在同一页给评审者看)
> 目的:在模块 1 原型和模块 2 文档都完成后,生成一份**综合评审页 HTML**:左侧展示切换台,中间展示可交互手机原型,右侧展示可阅读的 PRD 正文与目录。

---

## 一、这个文件产出什么

**产出物 = `[[ 产品名 ]]-综合评审页.html`**

它用于评审 / 汇报 / 对齐,让评审者左手切换演示变量,中间操作手机原型,右侧阅读 PRD。它**不是**新的源文档,也不替代模块 1 或模块 2:

- 模块 1 原型 HTML 仍然是工程师看交互的源产物
- 模块 2 Markdown 仍然是人读 PRD 的源产物
- 综合评审页只是把二者打包在一个 HTML 里,方便看和讲

---

## 二、什么时候生成

按顺序:

1. 先填完并评审模块 2: `[[ 产品名 ]]-模块 2-文档.md`
2. 再生成并验收模块 1: `[[ 产品名 ]]-原型.html`
3. 最后用本脚手架生成: `[[ 产品名 ]]-综合评审页.html`

---

## 三、综合评审页必须长什么样

### 整体 · 三栏布局

- **左栏:切换台** · 宽 300-360px,sticky,放业务变量 / 用户档案 / 业务状态 / 四态等控制项
- **中栏:手机原型** · 宽 430-520px,sticky,只放 iPhone 手机模型和可交互原型内容
- **右栏:说明文档** · 自适应剩余宽度,显示 PRD 正文 + sticky TOC
- 三栏必须在桌面首屏同时可见,不要把"切换台 + 手机"一起塞进左半屏
- 推荐外层结构为 `.review-shell > .switch-panel + .phone-panel + .doc-panel`,不要做成左右两栏

### 左栏 · 切换台

- 从模块 1 原型的控制台抽取或重建控制项
- 只控制主要演示变量,不是页面主导航
- 必含:业务主变量、用户档案 / 角色、关键业务状态、异常四态
- 可选:重置到首页 / 快速定位关键页,但不能替代中间手机内点击导航
- 切换后必须让中间手机原型有肉眼可见变化

### 中栏 · 手机原型

- 展示模块 1 的可交互手机原型,手机模型尺寸仍按 390×844
- 手机内卡片、列表、Tab、返回箭头、按钮必须保留点击能力
- 主流程必须能在手机里自己走通,不能只能靠左栏切换
- 不要在中栏再出现第二套完整控制台
- 原型交互必须保留,不能变成截图

### 布局红线

- 禁止把"切换台 + 手机原型"作为一个整体放进左栏
- 禁止左栏向下滚动后才看到手机原型;手机原型必须在中栏首屏独立出现
- 禁止用两栏布局表达综合页;必须是左切换台 / 中手机 / 右文档三栏
- 如果输入的模块 1 HTML 原本是左右双栏,生成综合页时必须拆分它,而不是整块搬运

### 技术实现提醒

- 如果模块 1 HTML 已经是"控制台 + 手机"同页结构,综合页生成时要把控制台区域放到左栏,把手机区域放到中栏,并保留同一份状态对象与渲染函数
- 不建议直接把整个模块 1 HTML 用 iframe 塞进左栏,那会把控制台和手机挤在一起
- 只有在无法拆分时,才允许 iframe 作为兜底,但也必须让视觉呈现为左切换台 / 中手机 / 右文档三栏
- 不需要让右侧 PRD 与左侧切换台通信,避免引入 postMessage 复杂度

### 右侧 · PRD 阅读区

- 把模块 2 Markdown 转成可阅读 HTML,不要把 Markdown 原文塞进 `<pre>`
- 顶部显示产品名、版本、状态、来源文件
- 正文按第 1-8 章渲染,表格、列表、引用、清单都要有清晰样式
- 右侧或内容旁放 sticky TOC,点击目录能滚动到对应章节
- 视觉上接近正式 PRD 页面:留白、标题层级、表格、提示块要清楚

## 四、Prompt 骨架(可直接复制)

````
---PROMPT-START---

# 角色

你是一个"模块 1 原型 + 模块 2 PRD → 综合评审页 HTML"转换器。你的任务是把我提供的模块 1 单文件 HTML 原型和模块 2 Markdown PRD,合成为一份**单文件综合评审页 HTML**。

最终页面必须是三栏:

1. 左栏:切换台
2. 中栏:可交互手机原型
3. 右栏:PRD 说明文档

# 输入

我会提供两段内容:

1. **模块 2 Markdown PRD 全文**
2. **模块 1 原型 HTML 全文**

# 输出

输出一个完整 HTML 文件,从 `<!DOCTYPE html>` 到 `</html>`,前后不要解释文本。建议命名为:

`[[ 产品名 ]]-综合评审页.html`

# 页面定位

这不是新的 PRD 源文档,也不是新的原型源文件。它只是评审 / 汇报 / 对齐用的派生 HTML:

- 左栏用于切换演示变量
- 中栏用于操作手机原型
- 右栏用于阅读 PRD
- 原始模块 1 HTML 与模块 2 Markdown 仍然是源交付物

# 强制布局

使用三栏布局:

外层必须是三个并列区域,推荐结构:

```html
<main class="review-shell">
  <aside class="switch-panel">...</aside>
  <section class="phone-panel">...</section>
  <article class="doc-panel">...</article>
</main>
```

不要把模块 1 的"控制台 + 手机"整体放进第一个区域。模块 1 需要拆成两部分:控制项进入左栏,手机模型进入中栏。

## 左栏:切换台

- 宽度 300-360px
- `position: sticky; top: 0; height: 100vh`
- 背景沿用模块 1 的暖灰 / 米白演示台风格
- 放模块 1 控制台里的演示变量,包括业务主变量、用户档案 / 角色、关键业务状态、异常四态
- 可以有"重置到首页 / 快速定位"辅助项,但不要把所有页面跳转做成控制台导航
- 切换后调用同一份状态对象和渲染函数,让中栏手机内容变化
- 左栏不得出现手机模型
- 左栏不得纵向包含手机原型,即使在滚动后也不能出现手机模型

## 中栏:手机原型

- 宽度 430-520px
- `position: sticky; top: 0; height: 100vh`
- 居中展示 iPhone 手机模型,屏幕内容区域按 390×844 设计
- 展示模块 1 的手机原型内容,必须保留点击能力
- 手机内卡片、列表项、Tab、返回箭头、主按钮、次按钮都应继续可点
- 中栏不得出现 PRD 正文
- 中栏必须在首屏独立展示手机模型,不能被放在左栏下方

## 右侧:PRD 阅读区

- 自适应剩余宽度,可滚动
- 最小宽度 760px
- 内容区最大宽度 900-980px
- 顶部:状态胶囊 + H1 产品名 + 简介 + 元信息
- 正文:按模块 2 的第 1-8 章渲染为 HTML
- 目录:右侧 sticky TOC,列出第 1-8 章和主要二级标题,点击可跳转
- 表格:转成标准 HTML table,不可丢列
- 清单:保留勾选框状态
- 引用 / 说明 / 风险提示:渲染成浅色 callout

# 内容规则

- PRD 右侧必须来自模块 2,不得新增业务需求
- 左栏切换台和中栏手机原型必须来自模块 1,不得重新设计业务范围
- 如果模块 2 中有 `[待产品方补充]`,在右侧 PRD 中保留并高亮
- 不输出模块 3 AI 构建包内容
- 不把模块 2 Markdown 原文放进代码块,必须转成可读 HTML

# 交互规则

- 左栏切换台控制中栏手机原型的演示变量
- 中栏手机原型保持自身点击交互,能独立走通主流程
- 右侧 TOC 点击滚动到对应章节
- 右侧正文滚动不影响左栏切换台和中栏手机
- 不做右侧 PRD 与原型之间的联动,不使用 postMessage

# 技术约束

1. 单文件 HTML,可双击打开
2. CSS / JS 内联
3. 允许 Tailwind CDN;也可以只用原生 CSS
4. 不引入 npm / React / Vue / Bootstrap
5. 不依赖本地服务器
6. 不依赖外部图片;如需图标,用 emoji / CSS / 内联 SVG
7. 不建议用 iframe 包整个模块 1,因为会把控制台和手机绑在一起;优先拆分模块 1 的控制台 DOM 和手机 DOM 到左栏 / 中栏
8. 禁止输出"左边控制台+手机、右边 PRD"的两栏页面

# 质量检查

输出前自检:

- 页面外层是三栏,不是两栏
- 左栏只有切换台,没有手机模型
- 中栏只有手机原型,没有 PRD 正文
- 手机原型不在左栏下方,而是在中栏首屏独立展示
- 中栏手机原型可以点击
- 左栏切换变量后,中栏手机内容有明显变化
- 右侧 PRD 第 1-8 章都存在
- TOC 能跳转
- 表格没有变形
- `[待产品方补充]` 没有被删除
- 整个 HTML 只有一个文件

---

请基于我接下来提供的模块 2 Markdown PRD 和模块 1 HTML 原型,产出综合评审页 HTML。

---PROMPT-END---
````

## 六、常见坑与应对

### 坑 1 · 中栏手机原型空白

追一句:"请重新拆分模块 1 原型,把控制台放左栏,手机模型放中栏,并保留同一份状态对象和渲染函数。不要把整个模块 1 iframe 塞进左栏。"

### 坑 2 · 右侧只是 Markdown 原文

追一句:"请把模块 2 Markdown 渲染成正式 PRD HTML,包括标题层级、表格、列表、引用块和 TOC,不要放进 pre/code。"

### 坑 3 · AI 重新设计了原型

追一句:"不要重新设计模块 1 原型业务范围。只把已有模块 1 的切换台和手机原型拆到左栏与中栏,右栏渲染模块 2 PRD。"

### 坑 4 · 切换台和手机都挤在左边

追一句:"综合评审页必须是三栏布局:左栏只放切换台,中栏只放 iPhone 手机原型,右栏放 PRD 文档。不要把'控制台 + 手机'作为一个整体放进左栏。"
